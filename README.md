# Train Travel

基于 Three.js 的程序化无限火车漫游模拟器。火车沿着动态生成的轨道行驶，穿越草原、森林、山脉、沙漠和湖泊等多种生物群落，配有完整的昼夜循环、天气效果和多视角相机系统。

![TypeScript](https://img.shields.io/badge/TypeScript-5.7-blue)

## 快速开始

```bash
npm install
npm run dev          # 启动开发服务器 (Vite)
npm run build        # 类型检查 + 生产构建
npm run typecheck    # tsc --noEmit
```

## 核心技术

### 无限轨道生成

轨道由 `TrackManager` 管理，采用分段式程序化生成策略：

- **分段结构**：每段约 100m 弧长，使用三次 Bezier 曲线表示
- **曲线连续性**：端点处通过 Hermite 插值保证 C1 连续（位置 + 切线匹配），转换公式为 `P1 = P0 + T0/3`，`P2 = P3 - T1/3`
- **前瞻生成**：始终保持火车前方 5 段轨道就绪
- **视锥裁剪**：火车后方超过 3 段且不在相机视锥内的段被回收释放
- **路径随机化**：使用 Simplex 噪声控制转弯角度（`noise2D(distance * 0.005, 0.5) * 0.4`），生物群落控制目标高程

**火车定位**使用 `(segmentIndex, localT)` 二元组表示，通过弧长参数化实现速度无关的匀速运动。车厢通过 `walkBack()` 沿轨道回溯固定弧长距离来定位。

### 轨道几何体

每段轨道包含四层几何体：

| 组件 | 实现方式 | 说明 |
|------|---------|------|
| 铁轨 | `TubeGeometry` × 2 | 双轨间距 1.2 单位，半径 0.06 |
| 道砟 | 手工 `BufferGeometry` | 梯形截面，手算法线避免 Frenet 旋转 |
| 路基 | 手工 `BufferGeometry` | 8m 宽梯形截面基础 |
| 枕木 | `InstancedMesh` | 间距 2 单位，单次绘制调用 |

法线计算使用水平向量 `(-tangent.z, 0, tangent.x)` 而非 Frenet 框架，避免了曲线扭转导致的光照异常。

### 地形系统

**高度计算**采用多层噪声叠加：
1. 宽尺度地形：`fbm(x * 0.004, z * 0.004, 3)`
2. 细节噪声：两层不同频率的 Simplex 噪声
3. 生物群落混合：加权平均基础高度和缩放

**轨道-地形交互**根据高度差分三种情况处理：

- **高度差 < 5m → 路堤**：轨道两侧 6m 内平坦，6-12m 五次 smoothstep 过渡（C2 连续），12-60m 渐变回自然地形
- **轨道高于地形 > 5m → 桥梁**：生成双柱桥墩、交叉支撑、护栏系统，柱间距 15 单位
- **轨道低于地形 > 5m 且持续 10m+ → 隧道**：`TubeGeometry` 管状隧道，半径 5 单位，带石拱门和门柱

### 生物群落

五种生物群落按固定序列循环，总周期 1000 距离单位：

```
草原(200) → 森林(200) → 山脉(150) → 沙漠(150) → 湖泊(150)
```

- **过渡混合**：边界 30 单位范围内进行颜色和高度的平滑插值
- **湖泊水面**：双层渲染（深水层 + 波浪表面层），程序化波浪位移动画

### 植被与场景物件

每段轨道生成 80 个植被对象 + 2 栋房屋 + 2 个信号灯，使用种子随机数（`seed = segmentIndex * 7919 + 42`）确保确定性。

- **松树**：圆柱树干 + 3 层锥体树冠
- **阔叶树**：粗树干 + 球形树冠
- **仙人掌**：圆柱体 + 两条手臂（仅沙漠）
- **岩石**：十二面体基底 + 1-2 颗卫星岩
- **房屋**：箱体墙壁 + 锥形屋顶 + 烟囱 + 门窗

植被分布与生物群落关联：森林区 70% 松树，草原区 35% 阔叶树，山脉区高处只有岩石，沙漠区仅有仙人掌和岩石。

### 天穹与昼夜循环

完整的昼夜系统，一个周期 180 秒（现实时间）：

**天空着色器（自定义 GLSL）**：
- 天空渐变色随时段变化（黎明暖色 → 白天蓝色 → 黄昏橙红 → 夜晚深蓝）
- 太阳盘面 + 辉光 + 大气散射（`smoothstep` + `pow` 组合）
- 月亮蓝色辉光（夜间可见）
- 程序化云层：4 层 Perlin 噪声，阈值密度渲染，随光照变色
- 800 颗星星，自定义着色器实现闪烁动画

**光照系统**：
- 方向光（太阳）：100 单位半径轨道运动，强度 0.2-1.5
- 环境光：强度 0.15-0.5
- 月光：太阳对侧，夜间强度 0.6
- 阴影：PCF 软阴影，仅白天开启（1024×1024 阴影贴图）
- ACES Filmic 色调映射，曝光 0.8-1.3
- 指数雾：`FogExp2`，颜色匹配天际线

### 相机系统

三种模式（C 键切换）：

| 模式 | 说明 | 特点 |
|------|------|------|
| 追踪相机 | 后方 12 单位 | 鼠标拖拽旋转/调整高度 |
| 侧视相机 | 侧方 10 单位 | 固定侧面视角 |
| 驾驶室相机 | 车内第一人称 | 前方 20 单位注视点 |

相机运动使用 `lerp(position, target, 0.05)` 进行平滑插值，避免抖动。

### 火车模型

**机车**：高精度程序化模型，包含锅炉、烟箱、6 个辐条轮（自定义轮辋+轮毂+8 辐条）、活塞连杆系统、驾驶室（含窗户）、排障器、缓冲器、走板、烟囱、蒸汽穹顶、安全阀、汽笛、前灯和煤水车。

**客车** × 2：箱体车身 + 弧形车顶 + 8 个窗户 + 端墙门 + 转向架（含板簧）+ 缓冲系统。

车速范围 0-400 km/h，使用 `lerp(speed, targetSpeed, 0.02)` 平滑加减速。

## 性能优化

| 策略 | 实现 |
|------|------|
| 实例化渲染 | 枕木使用 `InstancedMesh`，单次绘制调用 |
| 对象池 | 烟雾粒子预分配 30 个网格复用 |
| 视锥裁剪 | 视锥外轨道段及其附属物移除 |
| 几何体回收 | `dispose()` 清理几何体和材质，防止内存泄漏 |
| 阴影优化 | 仅白天启用阴影渲染 |
| 像素比限制 | `Math.min(devicePixelRatio, 2)` 防止高分屏性能问题 |

## 关键算法

| 算法 | 位置 | 用途 |
|------|------|------|
| Simplex 噪声 | `noise.ts` | 地形/云层程序化生成 |
| 分形布朗运动 (FBM) | `noise.ts` | 多层噪声叠加增加细节 |
| Hermite → Bezier 转换 | `trackSegment.ts` | 从切线创建平滑轨道曲线 |
| 弧长参数化 | `trackManager.ts` | 速度无关的匀速运动 |
| 五次 Smoothstep | `terrain.ts` | C2 连续路堤过渡 |
| 种子随机数 | `vegetation.ts`, `noise.ts` | 确定性程序化放置 |
| 视锥裁剪 | `trackManager.ts` | 屏幕外几何体移除 |
